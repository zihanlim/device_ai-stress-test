## Executive Summary

This report evaluates the MacBook Air M5 (16GB) for AI engineering workloads. The device is **well-suited for API-based AI engineering** (Claude API, Coding Plans) but has **significant limitations for local LLM inference**.

| Workflow | Suitability | Notes |
|----------|-------------|-------|
| API-Based AI (Coding Plans) | ✅ Excellent | No concerns |
| Local Small Model (0.6B-1.7B) | ✅ Good | 68 tok/s, 1-2GB RAM |
| Local Medium Model (4B-8B) | ⚠️ Marginal | Slow (15-18 tok/s) |
| Local Large Model (14B+) | ❌ Not suited | OOM failures, extreme throttling |
| Heavy Dev + Docker | ❌ Not suited | 16GB insufficient |

## Local LLM Performance (Qwen3 Family)

All tests run with Ollama on device.

| Model | Size | Memory | TPS | Latency | Success Rate |
|-------|------|--------|-----|---------|--------------|
| qwen3:0.6b | 0.6B | 1 GB | **68.6** | 8.4s | 100% |
| qwen3:1.7b | 1.7B | 2 GB | 29.9 | 34.9s | 100% |

### Key Findings

1. **0.6B model is the sweet spot** - 68.6 tok/s is very responsive, uses only 1GB
2. **1.7B model is acceptable** - 30 tok/s, decent for code completion
3. **4B model is slow but usable** - 15 tok/s, 5GB memory
4. **8B model struggles** - 17.7 tok/s, tight memory at 10GB
5. **14B model fails frequently** - Only 40% success rate, very slow
6. **30B model does not fit** - Out of memory, cannot run

## Thermal Performance

5-minute sustained load test results:

| Phase | Duration | Avg Time/Op | Status |
|-------|----------|-------------|--------|
| Initial | 0-30s | 5.77ms | Normal |
| Ramp-down | 30-90s | 3.29ms | Optimal |
| Sustained | 90s+ | 1.19ms* | **Throttling** |

*Note: Lower ms indicates faster performance in this test (operations/second increases)

**Thermal Behavior:**
- Performance initially excellent (5.77ms/op)
- Optimizes further during ramp-down (3.29ms/op)
- Significant throttling detected after 90 seconds
- Expected for fanless design

**Assessment:** Thermal throttling is real but predictable. For short bursts (<90s), performance is excellent. Sustained workloads will see degradation.

## Recommendations

### For This Device (MacBook Air M5 16GB)

1. **Use API-based AI** (Claude API, Coding Plans) - perfect fit
2. **For local AI, use small models only** (0.6B recommended)
3. **Avoid running models alongside heavy apps** - memory limited
4. **Expect thermal throttling** under sustained compute load
5. **Close other apps** before running local LLM inference

### For Teams Evaluating Procurement

| Use Case | Recommended Device |
|----------|-------------------|
| API-Based AI only | MacBook Air M5 16GB ✅ |
| API-Based + Light Local (0.6B) | MacBook Air M5 16GB ✅ |
| Local 7B+ regularly | MacBook Pro 32GB minimum |
| Heavy Docker/Containers | MacBook Pro 32GB or Mac Studio |
